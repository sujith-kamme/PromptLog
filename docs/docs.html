PromptLog Documentation

A lightweight, framework-agnostic prompt tracking library for LLM projects. Automatic logging, versioning, and run comparison — in three lines of code.

Installation

Clone the repository and install in editable mode:

git clone https://github.com/sujith-kamme/PromptLog.git
cd PromptLog
uv pip install -e ".[examples]"

To also install the web dashboard dependencies:

uv pip install -e ".[examples,ui]"
Requirements: Python 3.11+ • SQLite (bundled with Python) • uv (or use pip install -e directly)

Quick Start

Add prompt tracking to any LLM call in three lines:

import promptlog as pl

# 1. Initialize — all runs are grouped under this project
pl.init(project="my_chatbot")

# 2. Decorate any function that calls an LLM
@pl.track(model="gpt-4o", temperature=0.7)
def ask(question: str) -> str:
    response = openai.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": question}],
    )
    return response.choices[0].message.content

# 3. Call your function — PromptLog records everything automatically
answer = ask("What is the capital of France?")
print(answer)

Then inspect your runs from the CLI:

# List recent runs
promptlog ls --project my_chatbot

# View full details for a specific run
promptlog view abc12345 --project my_chatbot

# Launch the web dashboard
promptlog serve

pl.init()

Initialize PromptLog for your project. Must be called once before any @pl.track() calls. Creates or connects to a SQLite database and generates a unique session ID for this run. 

import promptlog as pl

config = pl.init(
    project="my_chatbot",
    storage_path=None,           # auto-resolved (see Storage Paths)
    feedback_mode="none",        # "none" or "end"
    default_model="gpt-4o",      # fallback for all @pl.track calls
    default_temperature=0.7,     # fallback temperature
    default_tags={"env": "prod"},# merged into every run's tags
    enabled=True,                # set False to disable all logging
)

Parameters

ParameterTypeDefaultDescription
projectstrrequiredProject name. All runs are grouped under this name.
storage_pathstr | NoneNoneExplicit path for the SQLite DB. If None, auto-resolved (see Storage Paths).
feedback_mode"none" | "end""none""end" triggers an interactive review prompt when the script exits. "none" is silent.
default_modelstr | NoneNoneFallback model name for all tracked functions.
default_temperaturefloat | NoneNoneFallback temperature for all tracked functions.
default_tagsdict | None{}Tags merged into every run's metadata.
enabledboolTrueSet False to disable all logging globally. Tracked functions still execute normally.

Returns

PromptLogConfig — the active configuration object (singleton).

Example: Disable logging in tests

import os
import promptlog as pl

pl.init(
    project="my_chatbot",
    enabled=os.getenv("ENVIRONMENT") != "test",
)

@pl.track()

Decorator that automatically logs every call to the decorated function as a Run. Captures the prompt (from arguments), output (return value), latency, errors, and all configuration. 

@pl.track(
    name="summarizer",              # override display name (default: function name)
    model="gpt-4o",                 # LLM model identifier
    temperature=0.3,                # sampling temperature
    max_tokens=500,                 # max tokens to generate
    top_p=0.9,                      # nucleus sampling
    top_k=50,                       # top-k sampling
    system_prompt="You are a helpful summarizer.",
    prompt_template="Summarize: {text}",  # auto-hashed for versioning
    tags={"task": "summarization"},       # arbitrary metadata
)
def summarize(text: str) -> str:
    response = openai.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": "You are a helpful summarizer."},
            {"role": "user", "content": f"Summarize: {text}"},
        ],
        temperature=0.3,
        max_tokens=500,
    )
    return response.choices[0].message.content

Parameters

ParameterTypeDefaultDescription
namestr | NoneNoneOverride the display name. Defaults to the function name.
modelstr | NoneNoneLLM model identifier (e.g., "gpt-4o", "claude-sonnet-4-20250514").
temperaturefloat | NoneNoneSampling temperature.
max_tokensint | NoneNoneMaximum tokens to generate.
top_pfloat | NoneNoneTop-p (nucleus) sampling parameter.
top_kint | NoneNoneTop-k sampling parameter.
system_promptstr | NoneNoneStatic system prompt text.
prompt_templatestr | NoneNoneRaw template before variable substitution. Auto-hashed (MD5, 8 chars) for versioning.
configdict | NoneNoneDict fallback for any parameters above. Lower priority than explicit args.
tagsdict | NoneNoneArbitrary metadata merged with global default_tags.

Config Priority

Parameters are resolved in this order (highest priority first):

  1. Explicit params on @pl.track(model="gpt-4o")
  2. The config dict: @pl.track(config={"model": "gpt-4o"})
  3. Defaults from pl.init(default_model="gpt-4o")
  4. None

Example: Using config dict

MODEL_CONFIG = {
    "model": "gpt-4o",
    "temperature": 0.7,
    "max_tokens": 1000,
}

@pl.track(config=MODEL_CONFIG, name="chat_agent")
def chat(user_message: str) -> str:
    # ... your LLM call
    pass
Note: @pl.track() only supports synchronous functions in v0.1. Decorating an async function raises NotImplementedError.

pl.log_prompt()

Explicitly record the rendered prompt for the current tracked run. Call this inside a @pl.track()-decorated function to capture the exact prompt sent to the LLM. 

@pl.track(model="gpt-4o", prompt_template="Translate {lang}: {text}")
def translate(text: str, lang: str) -> str:
    prompt = f"Translate to {lang}: {text}"
    pl.log_prompt(prompt)  # explicitly capture the rendered prompt

    response = openai.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": prompt}],
    )
    return response.choices[0].message.content
Auto-capture fallback: If log_prompt() is never called, PromptLog automatically joins all string arguments passed to the function as the prompt. Use log_prompt() when you need to capture the exact rendered prompt rather than raw arguments.
Error: Calling pl.log_prompt() outside a @pl.track()-decorated function raises RuntimeError.

Nested Tracking

When a @pl.track()-decorated function calls another tracked function, PromptLog automatically sets the parent_run_id — creating a hierarchy of runs. This is useful for agent pipelines and multi-step workflows. 

import promptlog as pl

pl.init(project="agent_pipeline")

@pl.track(model="gpt-4o", name="research_agent")
def research(topic: str) -> str:
    # ... calls LLM to gather information
    return "Research findings..."

@pl.track(model="gpt-4o", name="writer_agent")
def write(findings: str) -> str:
    # ... calls LLM to write an article
    return "Final article..."

@pl.track(name="pipeline")
def run_pipeline(topic: str) -> str:
    findings = research(topic)   # child run — parent_run_id set automatically
    article = write(findings)    # child run — parent_run_id set automatically
    return article

run_pipeline("quantum computing")

The CLI displays nested runs as a tree: 

$ promptlog ls --project agent_pipeline
# run_id   name             model    ms     scored  passed
# a1b2c3d4 pipeline         -        3200   no      -
# ├─ e5f6g7h8 research_agent gpt-4o  1500   no      -
# └─ i9j0k1l2 writer_agent   gpt-4o  1700   no      -

Auto-Capture

PromptLog automatically captures several values without any extra code:

Prompt

If pl.log_prompt() is not called, all string arguments passed to the function are joined and stored as the prompt.

Output

The function's return value is converted to a string and stored as the output.

Latency

Wall-clock execution time in milliseconds, measured from function entry to return.

Errors

If the function raises an exception, it's captured as "ExceptionType: message". The exception is re-raised after logging.

Parent Run

Nested @pl.track() calls automatically set parent_run_id using context variables.

Prompt Versioning

When you provide a prompt_template, PromptLog automatically generates a version hash (MD5, first 8 hex characters). Same template always produces the same version — making it easy to compare performance across prompt changes. 

# Version: "a3f8b2c1" (auto-generated from template hash)
@pl.track(
    model="gpt-4o",
    prompt_template="Summarize the following text in {n} bullet points:\n{text}",
)
def summarize_v1(text: str, n: int = 3) -> str:
    # ...
    pass

# Version: "d7e9f4a0" (different template = different version)
@pl.track(
    model="gpt-4o",
    prompt_template="Create a {n}-point summary of:\n{text}\nBe concise.",
)
def summarize_v2(text: str, n: int = 3) -> str:
    # ...
    pass

Compare versions with the CLI:

$ promptlog stats --project my_chatbot
# name          version   total  scored  passed  failed  avg_score
# summarize_v1  a3f8b2c1  50     30      25      5       0.83
# summarize_v2  d7e9f4a0  50     30      28      2       0.91

Feedback Mode

Set feedback_mode="end" to automatically trigger an interactive review prompt when your script exits. Only runs from the current session are shown. 

import promptlog as pl

pl.init(project="my_chatbot", feedback_mode="end")

@pl.track(model="gpt-4o")
def ask(question: str) -> str:
    # ... your LLM call
    return "answer"

ask("What is Python?")
ask("What is JavaScript?")

# When the script exits, PromptLog prompts:
#   ask: "What is Python?" → "answer"
#   [p]ass / [f]ail / [s]core / [n]ext / [q]uit?

The review loop shows each run in compact mode and accepts:

  • p — Mark as PASS (score=1.0)
  • f — Mark as FAIL (score=0.0)
  • s — Enter a custom score (0.0–1.0), optional label, and notes
  • n — Skip this run
  • q — Quit and save progress

promptlog ls

List logged runs for a project, rendered as a hierarchical tree with parent-child relationships.

promptlog ls --project <name> [--name <task>] [--unscored] [--failed] [--last N]

Options

OptionTypeDefaultDescription
--projectstrrequiredProject name.
--namestr—Filter by function/task name.
--unscoredflagFalseShow only unscored runs.
--failedflagFalseShow only runs with errors.
--lastint—Show last N runs.

Example

# List last 5 unscored runs
promptlog ls --project my_chatbot --unscored --last 5

# List only runs for a specific task
promptlog ls --project agent_pipeline --name research_agent

promptlog view

View full details for a specific run — prompt, output, configuration, and feedback.

promptlog view <run_id> --project <name>

Display Panels

  • Run Profile: Task name, project, model, temperature, latency, version
  • Prompt: Full prompt text (cyan border)
  • Output/Error: Markdown-rendered output or error message (red)
  • Feedback: Label, score, and notes (if scored)

Example

promptlog view a1b2c3d4 --project my_chatbot

promptlog stats

View aggregated pass rates and average scores per task and prompt version.

promptlog stats --project <name>

Output Columns

  • name — Function/task name
  • version — Prompt template hash
  • total — Total runs for this (name, version) pair
  • scored — Runs with feedback
  • passed — Runs with PASS/CORRECT/PROCEED or score ≥ 0.5
  • failed — Runs with FAIL or score < 0.5
  • avg_score — Average feedback score (0.00–1.00)

Example

$ promptlog stats --project my_chatbot
# name       version   total  scored  passed  failed  avg_score
# ask        a3f8b2c1  100    60      52      8       0.87
# summarize  d7e9f4a0  45     20      18      2       0.92

promptlog rescore

Manually set or update feedback for a specific run.

promptlog rescore <run_id> --project <name> [--pass | --fail | --score <val>] [--label <str>] [--notes <str>]

Options

OptionTypeDescription
--projectstrProject name (required).
--passflagMark as PASS (score=1.0, label="PASS").
--failflagMark as FAIL (score=0.0, label="FAIL").
--scorefloatCustom score between 0.0 and 1.0.
--labelstrCustom label (e.g., "PARTIAL").
--notesstrFree-text notes.

At least one of --pass, --fail, --score, or --label is required. If the run is already scored, you'll be prompted to confirm the overwrite.

Examples

# Quick pass/fail
promptlog rescore a1b2c3d4 --project my_chatbot --pass
promptlog rescore e5f6g7h8 --project my_chatbot --fail

# Custom score with notes
promptlog rescore a1b2c3d4 --project my_chatbot --score 0.75 --label "PARTIAL" --notes "Got the facts right but missed the tone"

Web Dashboard — Overview

The web dashboard is a single-page application built with Alpine.js and Tailwind CSS. Launch it with promptlog serve.

Layout

  • Top Navigation: Project selector dropdown, view toggle (Runs / Stats), export buttons (CSV / JSON)
  • Summary Cards: Total Runs, Scored, Passed, Failed, Pass Rate — displayed when a project is selected
  • Runs View: Filterable table showing all runs with tree indentation for nested runs. Click any row to open the detail modal.
  • Stats View: Aggregated metrics per (name, version) group — total, scored, passed, failed, avg score.
  • Run Detail Modal: Full prompt/output display, configuration panel, parent/child hierarchy links, feedback form, and delete button.

Filtering & Search

The runs view includes a filter panel above the table. All filters are live — changes take effect immediately.

FilterTypeDescription
SearchText inputFull-text search across prompt and output fields (case-insensitive).
Unscored OnlyCheckboxShow only runs without feedback.
Failed OnlyCheckboxShow only runs that raised exceptions.
VersionDropdownFilter by prompt template version hash.
ModelDropdownFilter by LLM model name.
Last NNumber inputShow only the last N runs.

Feedback & Scoring

Click any run row to open the detail modal. The right pane contains a feedback form:

  • Score: Numeric input (0.0–1.0, step 0.1)
  • Label: Dropdown with options: PASS, FAIL, PARTIAL, CORRECT, or a custom value
  • Notes: Free-text textarea for comments
  • Save Feedback: Persists to the database via the REST API

You can also delete a run from the modal — click "Delete Run", then confirm.

Export

Click the "Export CSV" or "Export JSON" button in the top navigation to download all runs for the current project as a file.

  • CSV: Downloads {project}_runs.csv with all run fields (prompt/output truncated to 80 chars).
  • JSON: Downloads {project}_runs.json with full run data.

Storage Paths

PromptLog stores data in SQLite databases. The database path is resolved in this order:

  1. Explicit path — If you pass storage_path to pl.init(), that path is used directly.
  2. Local directory — If a .promptlog/ folder exists in the current working directory, the DB is stored at ./.promptlog/{project}.db.
  3. Global directory — Otherwise, falls back to ~/.promptlog/{project}.db.
# Explicit path
pl.init(project="my_bot", storage_path="/data/prompts/my_bot.db")

# Local (create .promptlog/ in your project)
# $ mkdir .promptlog
pl.init(project="my_bot")  # → ./.promptlog/my_bot.db

# Global fallback
pl.init(project="my_bot")  # → ~/.promptlog/my_bot.db
Tip: Create a .promptlog/ directory in your project root to keep databases local and version-controllable.

Database

PromptLog uses SQLite with WAL (Write-Ahead Logging) mode for concurrent access.

Schema

TABLE runs (
    run_id         TEXT PRIMARY KEY,
    session_id     TEXT,
    parent_run_id  TEXT,
    name           TEXT NOT NULL,
    project        TEXT NOT NULL,
    prompt         TEXT,
    output         TEXT,
    config_json    TEXT NOT NULL,    -- PromptConfig as JSON
    latency_ms     REAL,
    feedback_json  TEXT,             -- FeedbackResult as JSON
    timestamp      TEXT NOT NULL,    -- ISO 8601 datetime
    error          TEXT
)

Migrations

The database schema is auto-migrated on startup. If you're upgrading from an older version, PromptLog automatically adds the session_id and parent_run_id columns if they don't exist.
